% =============================================================================
\chapter{App Shell}
\label{ch:app-shell}
% =============================================================================

The app shell is the frame that wraps every authenticated page: the
collapsible left sidebar, the top header, the main content region, the
status footer at the bottom, and a slide-up activity console. The same
components render on every route, so they are documented once here and
referenced throughout the rest of the guide.

\label{sec:app-shell-appshell}
% Component: app/AppShell.tsx

  \pagelement{Header region}{Top bar with page title / breadcrumb and user menu.}
  \pagelement{Main region}{The current route's page renders here, inside a scrollable container.}
  \pagelement{StatusFooter}{Always-visible single-line bar anchored to the bottom edge.}
  \pagelement{ActivityConsoleDrawer}{Slide-up panel summoned from the StatusFooter's Console toggle.}
  \pagelement{PageGuideBanner}{Per-route dismissible tip banner rendered just under the Header when applicable.}
\end{itemize}

\begin{itemize}

\end{itemize}

\begin{designnote}
Two nav catalogues share one \texttt{Sidebar} component. Which catalogue
is used is decided by the presence of a pipeline context: global routes
see \texttt{NAV\_GROUPS}, pipeline routes see
\texttt{buildPipelineNavGroups(code)}.
\end{designnote}

\section{ProtectedRoute}
\label{sec:app-shell-protected-route}
% Component: app/ProtectedRoute.tsx

\texttt{ProtectedRoute} wraps every authenticated route. Unauthenticated
visits redirect to \route{/login}; role-gated routes (anything under the
Admin group) additionally require the appropriate role or redirect to
the Dashboard.

\begin{designnote}
Route guards live above the page components so pages themselves can
assume a valid session. Unsupported-role access is treated as a soft
redirect, not a 403 screen.
\end{designnote}
