% =============================================================================
\chapter{Appendix B: Icon Reference}
\label{ch:appendix-icons}
% =============================================================================

All icons used across the app come from \texttt{@/tokens/icons} and are
a curated subset of \texttt{lucide-react}. Icon meaning is
context-specific in this app — the same glyph can mean different
things on different surfaces. This appendix groups icons by category
and annotates the meaning in the alphaN2N UI.

\begin{designnote}
A few glyph meanings are worth calling out up-front so the rest of the
reference makes sense:
\begin{itemize}
  \pagelement{\texttt{CheckCircle} / \texttt{XCircle}}{on review controls these always mean ``Approve'' / ``Reject'' — never ``complete'' / ``error''.}
  \pagelement{\texttt{Eye}}{means ``preview / open in modal'' — not ``watched''.}
  \pagelement{\texttt{Star}}{marks the hero variant or a favourite.}
  \pagelement{\texttt{Clapperboard} / \texttt{Film}}{\texttt{Clapperboard} is a scene; \texttt{Film} is a clip and, on buttons, the delivery / output affordance.}
  \pagelement{\texttt{Folder} vs \texttt{FolderKanban}}{\texttt{Folder} is a library bucket, \texttt{FolderKanban} is a project.}
\end{itemize}
\end{designnote}
