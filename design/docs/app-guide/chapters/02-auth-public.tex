Only two routes render outside the app shell: the login form and the
external review share page. Both are public entry points and have no
sidebar, header or footer chrome.

\section{External Review Page}
\label{sec:auth-external-review}
% Route: /review/share/$token
% Component: app/pages/ExternalReviewPage.tsx

The route at \route{/review/share/\$token} renders an asset review surface
keyed by a signed share token. It shows:

\begin{itemize}
  \pagelement{Share context bar}{Summary of what is being reviewed, who shared it, and the expiry.}
  \pagelement{Player / viewer}{Renders the asset in read-only mode.}
  \pagelement{Approve / reject controls}{Present only when the token carries review scope.}
  \pagelement{Invalid / expired state}{Rendered when the token fails validation.}
\end{itemize}

\begin{integritynote}
External share tokens carry a scope and an expiry. The page never loads
project data beyond what the token permits, so a leaked URL cannot be
used to escalate access.
\end{integritynote}

No screenshot is included here — the page only renders meaningful content

\end{designnote}
